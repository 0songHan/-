    5.2 Neo4j图数据库的介绍

    Neo4j 介绍¶

    学习目标¶

    • 了解Neo4j图数据库的简介、版本说明
    • 了解节点, 关系, 属性, 标签的有关概念
    • 掌握Cypher语句的基本命令和语法
    • 掌握py2neo工具对Neo4j图数据库的操作

    Neo4j 介绍¶

    • Neo4j是由Java实现的开源NoSQL图数据库. 自从2003年开始研发, 到2007年发布第一版. neo4j现如今已经被各行各业的数十万家公司和组织采用.
    • Neo4j实现了专业数据库级别的图数据模型的存储. 与普通的图处理或内存级数据库不同, neo4j提供了完整的数据库特性, 包括ACID事物的支持, 集群支持, 备份与故障转移等. 这使其适合于企业级生产环境下的各种应用.
    • Neo4j的版本说明:
      • 企业版: 需要高额的付费获得授权, 提供高可用, 热备份等性能.
      • 社区开源版: 免费使用, 但只能单点运行.

    Neo4j 图数据库概念¶

    • 节点
      • 节点是主要的数据元素, 节点通过关系连接到其他节点, 节点可以具有一个或多个属性 (即存储为键/值对的属性), 节点有一个或多个标签, 用于描述其在图表中的作用. 示例: Person>节点.
      • 可以将节点类比为关系型数据库中的表, 对应的标签可以类比为不同的表名, 属性就是表中的列.
    • 关系
      • 关系连接两个节点, 关系是方向性的, 关系可以有一个或多个属性(即存储为键/值对的 属性).
    • 属性
      • 属性是命名值, 其中名称(或键)是字符串, 属性可以被索引和约束, 可以从多个属性创 建复合索引.
    • 标签
      • 标签用于组节点到集, 节点可以具有多个标签, 对标签进行索引以加速在图中查找节点

    Neo4j 图数据库的安装¶

    • 这里主要介绍windows下Neo4j图数据库是如何安装的
    • 第一步: 安装JAVA SE 的 JDK
      • 下载地址: https://www.oracle.com/java/technologies/downloads/#java11 (注意: 在点击下载的过程中, 可能需要需要oracle账户, 直接注册登录即可下载)
    • 下载安装完之后, 打开windows终端, 输入java --version检查是否安装好
    • 第二步: 下载安装Neo4j
      • 从官网下载社区版本, 下载地址: https://neo4j.com/download-center/#community

    • 下载好后, 解压到自己想放的盘里（注意, 路径不要出现中文, 以免报错）. 例如: C:\Program Files (x86)\neo4j-community-4.4.18
    • 第三步: 对Neo4j进行环境变量配置

    • 在path路径中将变量添加进去

    第四步: 启动neo4j数据库

    • 浏览器页面展示

    Cypher介绍与使用¶

    1 create命令¶

    • 创建图数据中的节点
    • 演示: 
    # 创建命令格式: 
# 此处create是关键字, 创建节点名称node_name, 节点标签Node_Label, 放在小括号里面()
# 后面把所有属于节点标签的属性放在大括号'{}'里面, 依次写出属性名称: 属性值, 不同属性用逗号','分隔
# 例如下面命令创建一个节点e, 节点标签是Employee, 拥有id, name, salary, deptnp四个属性: 
CREATE (e:Employee{id:222, name:'Bob', salary:6000, deptnp:12}）
    • 效果

    2 match命令¶

    • 匹配(查询)已有数据
    • 演示: 
    # match命令专门用来匹配查询, 节点名称: 节点标签, 依然放在小括号内, 然后使用return语句返回查询结果, 和SQL很相似. 
MATCH (e:Employee) RETURN e.id, e.name, e.salary, e.deptno

MATCH (n) return n # 查询所有结点
    • 效果

    3 merge命令¶

    • 若节点存在, 则等效与match命令; 节点不存在, 则等效于create命令.
    • 演示: 

    MERGE (e:Employee {id:146, name:'Lucer', salary:3500, deptno:16})
    - 效果:

    • 然后再次用merge查询, 发现数据库中的数据并没有增加, 因为已经存在相同的数据了, merge匹配成功.
    • 演示: 

    MERGE (e:Employee {id:146, name:'Lucer', salary:3500, deptno:16})
    - 效果:

    4 使用create创建关系¶

    • 必须创建有方向性的关系, 否则报错.
    • 演示: 
    # 创建一个节点p1到p2的有方向关系, 这个关系r的标签为Buy, 代表p1购买了p2, 方向为p1指向p2
CREATE (p1:Profile1)-[r:Buy]->(p2:Profile2)
    • 效果:

    5 使用merge创建关系¶

    • 可以创建有/无方向性的关系.
    • 演示: 

    # 创建一个节点p1到p2的无方向关系, 这个关系r的标签为miss, 代表p1-miss-p2, 方向为相互的
MERGE (p1:Profile1)-[r:miss]-(p2:Profile2)
    - 效果:

    6 where命令¶

    • 类似于SQL中的添加查询条件.
    • 演示: 

    # 查询节点Employee中, id值等于123的那个节点
MATCH (e:Employee) WHERE e.id=123 RETURN e
    - 效果:

    7 delete命令¶

    • 删除节点/关系及其关联的属性.
    • 演示: 

    # 注意: 删除节点的同时, 也要删除关联的关系边
MATCH (p1:Profile1)-[r]-(p2:Profile2) DELETE p1, r, p2
    - 效果:

    8 sort命令¶

    • Cypher命令中的排序使用的是order by.
    • 演示: 

    # 匹配查询标签Employee, 将所有匹配结果按照id值升序排列后返回结果
MATCH (e:Employee) RETURN e.id, e.name, e.salary, e.deptno ORDER BY e.id

# 如果要按照降序排序, 只需要将ORDER BY e.salary改写为ORDER BY e.salary DESC
MATCH (e:Employee) RETURN e.id, e.name, e.salary, e.deptno ORDER BY e.salary DESC

    在Python中使用Neo4j¶

    1 连接 Neo4j 数据库:¶

    • 要通过python来操作Neo4j, 首先需要安装py2neo, 可以直接使用pip安装. 
    pip install py2neo
    • 在安装完成后, 直接调用py2neo即可
    # 导入必备的工具包
# 导入创建节点、关系的工具包
from py2neo import Graph, Node, Relationship
# 导入查询节点、关系的工具包
from py2neo import NodeMatcher, RelationshipMatcher

# 实例化图数据库
# 连接neo4j数据库, 输入地址、用户名、密码
graph = Graph('http://127.0.0.1:7474/', auth=('neo4j', '12345'))

    2 建立节点¶

    • 使用py2neo创建节点, 我们一般直接使用 Node 节点类来创建, 建立节点的时候需要定义节点的类型（label）以及节点的属性（property）,
    • 下面是代码演示: 
    # 定义节点1, 标签类别为Student, 属性: name:jack, age:18
node1 = Node("Student",name = "jack", age=18) # 注意: 这里不要加“label=”, 否则label会以属性的形式展示, 而非标签
# 定义节点2, 标签类别为Student, 属性: name:lucy, age:20
node2 = Node("Student",name = "lucy", age=20)
# 调用graph对象的create方法完成节点的创建
graph.create(node1)
graph.create(node2)

    3 节点关系的建立¶

    • 使用py2neo创建关系时, 我们一般使用 Relationship 关系类来创建, 建立关系的时候, 需要考虑关系的方向, 因此必须定义一个起始节点和一个结束节点. （注意: 如果起始节点和结束节点相同, 那么这个关系就是这个节点指向自己）.
    • 在节点关系建立的过程中, 如果之前没有创建过节点, 那么会默认创建节点
    • 下面代码演示: 
    # 实例化关系类对象
relation1 = Relationship(node1,'Friend',node2)
# 调用graph对象的create方法完成关系的创建
graph.create(relation1)

# 实例化关系类对象
relation2 = Relationship(node2,'Friend',node1)
# 调用graph对象的create方法完成关系的创建
graph.create(relation2)

    4 使用 NodeMatcher 查询节点¶

    • 通常查询节点, 我们会创建一个 NodeMatcher 对象, 然后调用 match 方法来指明要匹配哪种label的节点, 还可以使用where来进行条件筛选（有两种方法）.
    • 注意: 如果节点存在, 返回的结果是 NodeMatcher 的对象, 需要转化成Node对象, 可以用first方法取出符合条件的第一个节点, 或者转化成节点的list.

    • 下面代码演示1:

      • 不带where条件, 返回所有满足标签的节点: 
      # 实例化节点查询对象
node_match = NodeMatcher(graph)

# 用list方式返回节点列表
nodes = list(node_match.match('Student'))
# print(nodes)--->打印结果如下: 
# [Node('Student', age=18, name='jack'), Node('Student', age=20, name='lucy')]

# 用first方法取出符合条件的第一个节点
node = node_match.match('Student').first()
# print(node)--->打印结果如下: 
# (_10:Student {age: 18, name: 'jack'})

    • 下面代码演示2:

      • where条件的第一种写法: 把匹配的属性和值写成key=value的形式, 例如上面where(age='18'), 这种写法只能按照值是否完全一致来匹配, 不能按照值的大小来筛选, 如果写成where(age>'18'), 会报错. 
      # 实例化节点查询对象
node_match = NodeMatcher(graph)

# 用list方式返回节点列表
nodes = list(node_match.match('Student').where(age=18))
# print(nodes)--->打印结果如下: 
# [Node('Student', age=18, name='jack')]

# 用first方法取出符合条件的第一个节点
node = node_match.match('Student').where(age=18).first()
# print(node)--->打印结果如下: 
# (_10:Student {age: 18, name: 'jack'})

    • 下面代码演示3:

      • where条件的第二种写法: 模糊匹配, 可以把条件表达式写成一个字符串, 整体放在where语句中, 在这个字符串中, 可以用 _ 来代指匹配到的节点. 
      # 实例化节点查询对象
node_match = NodeMatcher(graph)

# 用list方式返回节点列表
nodes = list(node_match.match('Student').where(age>18))
# print(nodes)--->打印结果如下: 
# [[Node('Student', age=20, name='lucy')]

# 用first方法取出符合条件的第一个节点
node = node_match.match('Student').where(age>18).first()
# print(node)--->打印结果如下: 
# (_11:Student {age: 20, name: 'lucy'})


# 将NodeMatcher返回的结果转化为Node数据类型或者Node的list之后, 可以轻松访问Node属性, 如上面最后一例的结果, 访问其中第一个节点的name属性: 
print(node['name']) ----> lucy

    5 使用RelationshipMatcher查询关系¶

    • 通常查询关系, 我们会创建一个 RelationshipMatcher 对象, 然后调用 match 方法来匹配关系,

      • 查询关系的match方法有三个及以上参数:
        • 第一个参数: 节点的序列或则set, 可以为None表示任意节点均可
        • 第二个参数: 指定关系的类型, 可以为None表示任意类型的关系均可
        • 第三个参数: 要匹配的属性, 写成key=value形式
      • 注意: match方法的返回值是RelationshipMatcher对象, 需要通过first转化成Relationship数据结构, 或者转化为 list .

    • 下面代码演示1,

      • 比如想要查询“jack”节点的所有关系. 先查询节点, 再根据节点查关系, r_type=None表示任意关系类型均可. 
      #为了方便观察, 因为之前只有一种关系, 所以我们可以再多创建几种关系
node1 = node_match.match('Student').where(name='jack').first()
node3 = Node("Location",name = "America")
node4 = Node("Sports", name='football')
relation3 = Relationship(node1,'Country',node3)
graph.create(relation3)
relation4 = Relationship(node1,'Hobby',node4)
graph.create(relation4)

      # 实例化关系查询对象
relation_match = RelationshipMatcher(graph)

node1 = node_match.match('Student').where(name='jack').first()
relationship = list(relation_match.match([node1], r_type=None))
# print(relationship) ---> 打印结果如下
"[
Hobby(Node('Student', age=18, name='jack'), Node('Sports', name='football')),        Country(Node('Student', age=18, name='jack'), Node('Location', name='America')),     Friend(Node('Student', age=18, name='jack'), Node('Student', age=20, name='lucy'))
]"

    • 下面代码演示2:

      • 比如想要查询“jack”和“lucy”关系. 两个节点的顺序表示要匹配的关系方向. 图中“jack”和“lucy”节点之间的朋友关系是双向的, 但是查询结果只给了一条
      # 实例化关系查询对象
relation_match = RelationshipMatcher(graph)

node1 = node_match.match('Student').where(name='jack').first()
node2 = node_match.match('Student').where(name='lucy').first()

    8 直接执行Cypher语句（run）¶

    • NodeMatcher 和RelationshipMatcher 能够表达的匹配条件相对简单, 更加复杂的查询还是需要用Cypher语句来表达. Py2neo本身支持执行Cypher语句的执行, 可以将复杂的查询写成Cypher语句, 通过graph.run方法查询, 返回的结果可以转化为 pandas.DataFrame 或者 pandas.Series 对象, 从而和其他数据分析工具无缝衔接. 
    cypher_ = "MATCH (n: Student) WHERE n.age > 16 RETURN n.name AS Name, n.age AS Age"
df = graph.run(cypher_).to_data_frame()
# pythonprint(df)--->打印结果如下: 
   Name  Age
0  jack   18
1  lucy   20

    9 使用schema查询图中节点和关系类型有哪些¶

    • 查看节点的类型用graph.schema.node_labels.

    • 查看关系的类型用graph.schema.relationship_types, 它们的返回值类型都是 frozenset , 是不能增删元素的集合. 

      # 查看所有节点类型
labels = graph.schema.node_labels
# print(labels)---> 打印结果如下: 
# frozenset({'Sports', 'Student', 'Location'})
# 查看所有关系类型
relation_types = graph.schema.relationship_types
# print(relation_types)---> 打印结果如下: 
# frozenset({'Country', 'Friend', 'Hobby'})
